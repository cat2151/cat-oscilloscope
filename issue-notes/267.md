# issue demo-libraryの処理が遅すぎる、処理時間が通常の30倍以上かかっている #267
[issues #267](https://github.com/cat2151/cat-oscilloscope/issues/267)

# 詳細
- wasmの処理が異常に遅い。16ms以内で完了することが確認済みである処理に、500msかかっている
    - メインdemoのLoad WAV File機能では、非常に高速に16ms以内に処理完了していることが明確にわかっている
    - メインdemoのLoad WAV File機能と同じ使い方をすべきだが、それができていない可能性がある
        - できているか調査すること
        - 現実に、Load WAV File機能のとき16ms以内で高速動作し、demo-libraryのとき500msで超低速動作しているのがはっきり事実としてわかっている。なので、それぞれ呼び出し方が違ってしまっているのは明白である。その違いを調査すること
        - もし「同じ使い方がそもそもできない」のであれば、それはそれで設計不備である
            - どう設計修正すればよいか調査すること
            - もし修正した結果、「どっちも500msかかる遅いシステム」にエンバグするのは致命的にNGなので、そうならないよう検討すること
    - 例えば、計測方法を検討すること。例えば、現在、1frameの処理に500msもかかってしまっているので、console.logに、「どこが重たいか」を処理時間の大きそうな場所に処理時間console.logを仕込んでおくのでも十分そう。例えば、demo-libraryは、そもそもヘッドレスブラウザですべてのtestを実施できるので、現在でもagentが十分にlogとtestを使って原因調査ができる考え。
    - もしヘッドレスブラウザで計測できなかった場合は、生成を中断して、失敗を報告すること
